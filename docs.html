Docker Installation (Recommended)

The quickest way to get Vault1337 running. Default credentials are intentionally weak — change them immediately.

Step Instruction
Install Docker See the official Docker docs for your OS
Pull the image docker pull vault1337/vault1337:latest
Run the container docker run -p 8000:8000 vault1337/vault1337:latest
Log in with the default credentials Username: admin
Password: changeme123

http://localhost:8000
Change your password immediately http://localhost:8000/admin
Add API keys (staff only) Navigate to: Admin > API Keys
(or http://localhost:8000/admin/keys)

Add keys for the best experience:
- VT_KEY — VirusTotal API key
- MALWARE_BAZAAR_KEY — Malware Bazaar API key
- ABUSEIPDB_KEY — AbuseIPDB API key
- SPUR_KEY — Spur Context API key
- SHODAN_KEY — Shodan API key
You're ready to go! Start statically analysing malware safely at http://localhost:8000

Manual Installation — Ubuntu 24.04

Step Instruction
1. Update your system sudo apt update && sudo apt upgrade -y
2. Install system dependencies sudo apt install -y git python3-venv python3-pip libimage-exiftool-perl libmagic1 nodejs npm
3. Clone the repository cd /var/www # or wherever you want it
git clone https://github.com/DanDreadless/Vault1337/
cd Vault1337/
4. Create required directories and copy env file mkdir -p sample_storage
cp .env.sample .env
5. Create a Python virtual environment and install dependencies python3 -m venv env
source env/bin/activate
pip install -r requirements.txt
6. Generate a Django secret key python3 -c 'from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())'
7. Edit your .env file nano .env

SECRET_KEY='paste-your-generated-key-here'
DEBUG=False

# Optional — enables full feature set:
VT_KEY='your-virustotal-api-key'
MALWARE_BAZAAR_KEY='your-malware-bazaar-api-key'
ABUSEIPDB_KEY='your-abuseipdb-api-key'
SPUR_KEY='your-spur-api-key'
SHODAN_KEY='your-shodan-api-key'
8. Run database migrations python3 manage.py makemigrations
python3 manage.py migrate
9. Create a superuser python3 manage.py createsuperuser
10. Build the React frontend cd frontend
npm install
npm run build
cd ..
11. Collect static files python3 manage.py collectstatic --noinput
12. Run the development server source env/bin/activate
python3 manage.py runserver

Open: http://127.0.0.1:8000

Notes

  • For production, serve with Gunicorn behind NGINX. Set DEBUG=False and configure ALLOWED_HOSTS in .env.
  • For a PostgreSQL database, add DATABASE_URL to your .env and install psycopg2-binary.
  • Sample files are stored in sample_storage/ and referenced by SHA256 hash — original filenames are never written to disk.
  • YARA rules are stored in vault/yara-rules/ as .yar files and can be managed via the web UI.
  • API keys can be updated at any time from the Admin > API Keys page (staff accounts only).
